### 🛠️ news-collector 能干啥？
简单说：你说需求，它去抓新闻，你收简报。
比如这些场景：
"帮我看看今天科技圈有啥大事"
"整理下本周财经热点，输出表格"
"搜一下'新能源'相关的新闻，不限分类"
Agent 接到指令后会自动：

✅ 判断要查哪些分类、要不要关键词过滤

✅ 从多个RSS源抓取、解析、去重

✅ 按热度/时间排序，生成结构化输出

✅ 还能记住你的偏好，越用越懂你最关键的是：后台默默干活，不打扰你干正事 💪

### 🧠 三层架构设计news-collector 也遵循 Skill 经典三层设计：
✅ 元信息层 —— 声明支持6大分类（财经/科技/娱乐/体育/国际/国内）

✅ 指令层 —— get_news、search、categories、export 四个核心指令

✅ 资源层 —— scripts/ 负责抓取解析，reference/ 提供分类定义和源配置Agent 读完 skill.md 就知道："哦，这是个专业的新闻小助手" ✨

## 🔍 为啥要做这个 Skill？ 
你可能会想：RSS订阅不香吗？新闻API不是大把？理想很丰满，现实很骨感：

❌ RSS源分散在几十个网站，格式乱七八糟，解析起来头大

❌ 同一事件被10家媒体重复报道，刷半天都是同一句话

❌ 科技频道里混着广告，财经新闻里夹着软广，人工筛选太费眼而普通 Agent 工具面对新闻场景，常常一脸懵：

⚠️ 只能抓单点信息，做不到跨源聚合

⚠️ 不懂分类逻辑，返回结果杂乱无章

⚠️ 输出格式不统一，还得自己二次加工
于是我决定：
做个专门懂新闻的 Skill，让 Agent 也能当个合格的新闻编辑 🎯

### 🚀 怎么用上它？
超简单，对你的 Agent 说：
帮我安装 news-collector skill，仓库地址是 https://github.com/yourname/news-collector
Agent 会自动下载、检查环境、注册指令。需要的环境：
Python 3.8+
requests、feedparser 库
装好后直接开用：
"帮我获取今天的科技新闻""搜索'人工智能'相关资讯"它就会开始工作啦 🎉
